# Account Workspace V3

## 定位

TZ 是多交易模式的执行纪律、账户记录和复盘工具，而不是行情终端、选股系统或自动交易系统。当前默认模式为五日线回踩，并预留模式2、模式3的规则框架。

## 数据边界

账户资金实体必须带 `mode`，并在模拟训练与实盘记录之间完全隔离。交易方法实体必须带 `mode + strategy_id`，并在不同交易模式之间隔离：

- 交易流水；
- 当前模式持仓监控与可卖数量展示；
- 延后处理决定；
- 持仓备注；
- 通知；
- 日、周、月复盘；

账户设置、初始资金、现金、总资产、账户总持仓、手续费、行情源和同花顺手工对账数字按 `mode` 保存，由同一账户下的各交易模式共享。

## 不接入的内容

- 自动选股、股票池、成交额前 20 排名；
- 盘中全市场扫描；
- 实时行情源；
- 券商登录、自动同步或自动下单；
- 自动荐股与买卖建议。

## 复盘模型

每条复盘记录包含四项：

1. `planAndBasis`：计划与交易依据；
2. `executionAndDeviation`：执行与纪律偏差；
3. `resultAndEmotion`：结果归因与情绪复盘；
4. `improvementAndNextPlan`：改进与下一交易日硬计划。

复盘页同时展示资金变化、资产曲线、已实现盈亏、浮动盈亏、手续费和违规类型统计。

## 持仓计算

持仓由当前账户交易流水按时间顺序推导：

- 买入增加批次；
- 卖出按 FIFO 扣减；
- 当日买入数量计入 T+1 锁定；
- 可卖数量只包含买入日期早于当前日期的剩余批次；
- 最近一笔成交价作为无实时行情时的参考价格；
- 同花顺手工对账只改变展示的期末数字，不修改交易流水。
